# Interview Talking Points

- The shell separates sessions, conversation, and context/tool state so users can reason about the agent.
- Agent modes help communicate intent and risk: Ask is low risk, Execute requires approval.
- RAG cards turn hidden model context into visible evidence.
- Tool timelines make MCP-style execution understandable and auditable.
- Approval cards create a human-in-the-loop boundary before data-changing actions.
- Execution states prevent ambiguous loading behavior and support recovery UX.
